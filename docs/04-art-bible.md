# 《牛马大逃杀》美术指导手册 / Art Bible

## 1. 视觉定位

“严肃公司文书包裹下的荒诞像素办公室战争。”

局内是深色、压抑、低饱和的开放办公区；武器、危险、掉落和角色身份用少量高饱和色爆发。局外 UI 则像盖满公章的纸质文件，形成“公司看起来正规，发生的事完全失控”的反差。

## 2. 视觉支柱

### 2.1 小尺寸也必须读得懂

单位在 640×360 逻辑画面中非常小，轮廓、朝向、威胁类型优先于细节。禁止只靠细微纹理区分敌人。

### 2.2 办公室物件武器化

桌子、键盘、订书机、马克杯、碎纸机、会议激光笔、奶茶、红头文件都应保留现实形状，再夸张成战斗用途。喜剧来自“熟悉物件做不熟悉的事”。

### 2.3 公司制度视觉化

裁员红线、考核范围、会议圈、举报标记和审批状态必须像制度标识：红线、印章、黄黑警示、表格、倒计时、打钩/打叉，而不是泛用魔法特效。

### 2.4 AI 武器不用 logo 也能识别

每把武器依靠固定颜色、弹道节奏和形状识别。避免使用官方 logo、界面或吉祥物；文字名称只出现在 HUD 和图鉴。

## 3. 基础色板

### 3.1 核心 UI/世界色

| 用途 | 色值 | 规则 |
|---|---|---|
| 深墨轮廓 | `#14161d` | 角色、家具、UI 硬边；避免纯黑吞细节 |
| 办公室底色 | `#101319` | 页面与画布外背景 |
| 纸张 | `#f2efe6` | 菜单、审批单、结算；略暖，不用纯白 |
| 暗纸 | `#e4dfd0` | 次级纸面、禁用态 |
| 工牌黄 | `#ffcf33` | 经验、选中、融合、关键提示 |
| 裁员红 | `#ff4f4f` | 圈外、危险、生命损失 |
| 公章红 | `#d43a2f` | 文书印章、传说名称、制度强调 |
| 国产青绿 | `#4ec9a0` | 队别标签，不作为唯一国别信息 |
| 硅谷蓝 | `#6aa3ff` | 队别标签、护盾/设备色 |
| 模组紫 | `#b665ff` | 蒸馏、Boss、异常技术 |
| 安全绿 | `#7ee08a` | 治疗、成功、保护 |
| 警示橙 | `#ff9440` | 精英、事件、次级危险 |

### 3.2 武器色

每个主武器保持唯一主色，所有弹体、芯片、HUD 图标和融合预览复用该色。白色武器必须加深色描边；黄色武器避免与经验/选中态混淆，应通过弹体形状区分。

### 3.3 色盲规则

- 红圈：红色 + 虚线边界 + 屏幕边缘箭头 + 圈外暗红晕。
- 治疗：绿色 + 十字/上升符号。
- 护盾：蓝色 + 六边形/环形边框。
- 传说：公章红/白金 + 五格全亮 + “传说”文字。
- 不允许只用颜色区分标准/Pro/Ultra，需增加 1/2/3 道外圈或角标。

## 4. 像素与分辨率规范

- 逻辑画布：640×360，16:9。
- 世界地砖：32×32 网格。
- 当前运行时工人精灵：16×20，5 帧。
- 当前 Boss 运行时精灵：24×28，2 帧。
- 当前展示用原生资源：工人 160×72 横向走路序列；Boss 98×106 双帧。
- 道具基础尺寸：约 10—12 px。
- 所有局内像素资源使用整数坐标、最近邻缩放，不做亚像素旋转模糊。

建议统一两套资产层级：

1. `runtime`：16×20/24×28 等低分辨率，服务战斗可读性。
2. `portrait/native`：菜单、结算和宣传图使用，允许更高分辨率。

禁止把高分辨率角色直接缩成局内小人；应单独绘制 runtime sprite。

## 5. 角色设计

### 5.1 玩家牛马

当前形象：白衬衫、深色裤、牛角、工牌黄识别色。轮廓必须同时读出“上班族”和“牛马”：

- 牛角在 1× 缩放下仍形成左右突起。
- 白衬衫与深色地板形成高对比。
- 玩家服装或脚下标记固定使用工牌黄，不随随机衬衫变化。
- 移动帧优先表现腿和肩部摆动，头部保持稳定便于瞄准。

建议动画：待机 2 帧、四向移动各 4—6 帧、受击 1 帧、倒下 4 帧、融合 4 帧。当前横向翻转可继续承担左右方向；上下方向可用身体遮挡变化表达。

### 5.2 普通同事

共享牛马骨架，以衬衫色、发型/角型、工牌和小配件区分。AI 性格应可视化：

- 卷王：红领带或上扬眉、偏暖色、移动时前倾。
- 普通：中性色。
- 摸鱼：松领带、咖啡、偏冷色、游走时姿态放松。

避免仅靠名字区分 19 名同事。

### 5.3 老板

当前高分辨率形象为黑西装、红领带、墨镜、夸张上身。设计目标是“体面、昂贵、危险”：

- 体型约普通单位 1.6—2 倍。
- 黑西装用冷灰高光与地板分离。
- 红领带与紫色蒸馏光形成双识别色。
- 半血狂暴时增加领带飘动、紫红轮廓和地面脉冲，不只改变弹幕数量。
- 反向蒸馏时显示玩家到 Boss 的紫色数据闪电和被偷能力名称。

### 5.4 HR 与精英

- HR：灰制服、文件夹/工牌、冷蓝灰；不要与普通同事只差名字。
- Tier 1 精英：紫色技术事故边框。
- Tier 2 小 Boss：橙色姓名条、独立轮廓和边缘箭头。
- 对齐守卫正面无敌时必须显示清晰盾面方向。

## 6. 怪物视觉语法

| 行为 | 轮廓/图形 | 预警 | 命中反馈 |
|---|---|---|---|
| 基础追击 | 圆/方块小体型 | 无或短红闪 | 小红粒子 |
| 高速冲刺 | 尖角、前倾 | 直线轨迹 + 0.5 秒蓄力 | 拉长残影 |
| 远程射击 | 顶部天线/信封口 | 枪口闪 + 弹道色 | 减速/诅咒图标 |
| 护盾 | 外层粗环/齿轮 | 护盾裂纹随耐久 | 破盾黄字 + 1.2 秒硬直 |
| 分裂 | 双层轮廓 | 死亡前抖动 | 纸张向外散开 |
| 复活 | 撤回箭头 | 尸体蓝色倒带 | “消息撤回中” |
| 光环 | 固定地面圆环 | 环边图标重复 | 玩家脚下状态标记 |
| 高闪避 | 半透明/断续轮廓 | 闪烁节奏 | “已读不回”残影 |
| 据点增益 | 静止大体块 | 连线到被强化单位 | 连线断裂 |
| 控制/眩晕 | 日历、会议、锁 | 黄圈 → 红圈 | 头顶锁/星号 |

同一行为跨怪物复用视觉语法，让玩家能迁移知识。

## 7. 场景美术

### 7.1 总体基调

深夜办公室：冷蓝灰地板、黑色边界、少量显示器青光、绿植与饮水机作为低威胁地标。背景不应比投射物鲜艳。

### 7.2 区域主题

建议区块化后建立五个部门：

- 开放工区：桌阵、显示器、文件堆，基准战斗区。
- 会议区：玻璃/长桌、投影幕，较长视线和狭窄入口。
- 茶水间：饮水机、咖啡、补给概率高，空间较开放。
- 服务器间：冷蓝灯、机柜走廊，远程战和弹射更危险。
- 老板办公室：木色/暗红、地毯、巨大办公桌，终局视觉锚点但不固定 Boss 出生。

### 7.3 家具

- 桌子是主要实体障碍，碰撞轮廓必须与可见桌面一致。
- 绿植和饮水机若无碰撞，应降低视觉重量，避免玩家误判。
- 家具底部需有 1—2 px 接地阴影。
- 可破坏物与不可破坏物必须有不同边缘语言；当前均不可破坏，不要画裂纹。

## 8. UI 风格

### 8.1 文书系统

菜单和弹窗基于纸张、公章、红头编号、虚线分隔、表格、签字和审批按钮。每种界面对应文书：

- 开始：全员红头文件。
- 升级：晋升审批单。
- 暂停：带薪休息申请。
- 死亡：离职证明。
- 胜利：优秀员工证书。
- 无尽失败：加班记录。
- 商店：员工福利采购单。
- 云存档：档案室/电子归档单。

### 8.2 HUD 层级

- 左上：生命、经验、等级、构筑与负面状态。
- 中上：存活人数、波次/缩圈、事件公告、Boss 条。
- 右上：小地图、时间、击杀、金币、开火模式。
- 下中：主动技能与当前主武器。

移动端允许缩放/简化：技术模组列表折叠为图标，击杀播报最多 3 条，小地图缩小，非关键提示自动消失。

### 8.3 字体

- 正文：系统中文无衬线，优先 PingFang SC / Microsoft YaHei / Noto Sans CJK SC。
- 数据：等宽字体。
- 最小局内字号建议 10 px 逻辑像素；当前 6—9 px 文本只适用于近距离标签，不用于关键规则。
- 标题不使用花体，靠字距、粗体和印章建立风格。

## 9. 特效

- 伤害：小型方形粒子，避免烟雾遮挡。
- 暴击：橙色更大数字 + 短促闪光。
- 融合：对应传说色的 360° 粒子爆发、震屏、名称停留 1.6 秒。
- 缩圈：红色虚线、圈外半透明填充、屏幕边缘红晕；收缩目标圈用白色细线。
- 蒸馏：紫色折线闪电、能力名、Boss 外轮廓脉冲。
- 诅咒：固定颜色滤镜只能作为辅助，必须配 HUD 标签和单位头顶符号。
- 震屏上限保持克制；提供关闭开关。

## 10. 音画协同

- 升级、拾取、融合、Boss、红圈、拒绝操作各有独立短音型。
- 高危预警音必须在伤害前至少 0.5 秒出现，且不被 BGM 掩盖。
- 同类机制复用音色：制度/警报用打卡机与提示音，AI/蒸馏用数字噪声，办公室物件用拟音。
- 低血量心跳不得频繁到造成听觉疲劳，暂停/弹窗时应停止延时音符。

## 11. 资产命名与交付

```text
category_subject_variant_state_direction_frame.ext

char_worker_player_walk_e_00.png
char_boss_default_enrage_00.png
mob_email_urgent_attack_00.png
fx_zone_warning_ring_00.png
ui_icon_status_overfit.png
```

交付要求：

- PNG 使用透明背景，禁用颜色配置导致的插值。
- sprite sheet 附 JSON manifest：帧宽、高、数量、锚点、碰撞盒、帧率。
- 角色锚点统一为脚底中心；投射物为图形中心；地面区域为圆心。
- 每个资产提供 1× 原图和至少 8× 最近邻预览。
- 不将品红占位色 `#ff00ff` 留在最终导出图；它仅用于运行时换色遮罩。

## 12. 美术验收清单

- 1× 逻辑分辨率下能否辨认玩家、Boss、精英和主要威胁？
- 黑白截图中轮廓是否仍可区分？
- 去掉文字后，主武器弹道能否被识别？
- 危险预警是否早于伤害，且不会被家具遮挡？
- UI 是否保持纸张/公章语言，没有混入通用霓虹手游面板？
- 移动端最小字号、按钮热区和 HUD 遮挡是否合格？
- 是否避免未经授权的品牌 logo 与官方视觉复刻？
